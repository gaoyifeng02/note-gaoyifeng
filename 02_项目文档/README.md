# 服务文档索引

> 每个服务按 **功能说明 / 原理架构 / 接口与交互 / 部署运维** 四个维度独立梳理，基于源代码深度分析生成。
> 核心文档部分为针对某些核心实现，单独深入生成。

---

## 文档规范

每个服务目录下包含 4 个文档和一个目录：

| 文件            | 维度    | 内容要点                        |
| ------------- | ----- | --------------------------- |
| `01_功能说明.md`  | 功能层面  | 服务定位与职责、核心功能列表、支持的业务场景      |
| `02_原理架构.md`  | 原理层面  | 架构设计、模块结构、核心流程、关键数据结构与算法    |
| `03_接口与交互.md` | 输入输出层 | 对外API、上下游依赖、MQ/DB交互、数据格式与协议 |
| `04_部署运维.md`  | 部署运维层 | 编译打包、部署方式、配置项、依赖库与外部服务、调试方法 |
| 05_核心文档       | 核心原理  | 对于核心原理进行细致划分，细致解析           |
