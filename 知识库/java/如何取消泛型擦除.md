~~~
  private static final TypeReference<HashMap<String, Object>> MAP_TYPE_REF = new TypeReference<>() {};                                                                       
                                                                  
  这是什么？
  - TypeReference<> 是 Jackson 库的抽象类，用于保留泛型类型信息
  - {} 表示创建一个匿名子类
  - <> 是钻石操作符（Java 7+），编译器会自动推断泛型类型

  为什么需要这样写？
  - Java 的泛型有类型擦除，运行时 HashMap<String, Object> 会变成 HashMap
  - 匿名子类会在字节码中保留泛型信息，Jackson 才能正确反序列化

  Java 8 写法对比：
  // Java 8 也支持，但很多人不知道
  new TypeReference<HashMap<String, Object>>() {}
~~~
