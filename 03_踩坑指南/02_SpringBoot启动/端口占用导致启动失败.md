# 端口被未关闭的 Java 进程占用导致启动失败

## 问题描述

Spring Boot 启动时报错：

```
Web server failed to start. Port 8080 was already in use.
```

## 原因

之前通过后台方式启动的 Spring Boot 进程未被干净地终止，端口仍然被占用。在 Git Bash 中使用 `kill $PID` 可能无法有效终止 Windows 上的 Java 进程。

## 解决方案

**Windows 环境下查找并强杀占用端口的进程：**

```bash
# 查找占用端口的进程 PID
netstat -ano | grep 8080

# 强制终止进程
taskkill /F /PID <PID>
```

## 建议

- 测试完启动后，确认进程已被终止再进行下一次启动
- 也可以在启动时使用不同端口避免冲突：`--server.port=8081`
