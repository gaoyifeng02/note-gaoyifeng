# Git 提交规范

## 提交信息格式

```
[类型] 简短描述;
```

## 类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| `add` | 新增功能、文件、依赖等 | `[add] 新增用户注册接口;` |
| `update` | 修改已有功能、优化逻辑、升级依赖等 | `[update] 优化订单查询性能;` |
| `fix` | 修复 Bug | `[fix] 修复登录超时异常;` |
| `delete` | 删除文件、移除无用代码、移除依赖等 | `[delete] 移除废弃的工具类;` |

## 规则

1. 类型必须从 `add`、`update`、`fix`、`delete` 中选择，使用方括号包裹
2. 描述部分使用中文，简明扼要说明本次提交的内容
3. 每条提交信息以分号 `;` 结尾
4. 一条提交信息只描述一件事，不要混合多个无关改动
5. 涉及多个改动时，拆分为多次提交

## 多条改动示例

当一次提交包含多个相关改动时，用顿号分隔：

```
[add] 新增支付回调接口、新增支付状态枚举;
[fix] 修复订单金额计算错误、解决并发扣款问题;
[update] 优化用户查询逻辑、升级mybatis-plus版本;
```

## 反例（禁止）

```
❌ 修改了代码
❌ fix bug
❌ [add]新增功能新增接口新增配置
❌ update了一些东西;
```
