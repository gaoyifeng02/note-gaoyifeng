# 学习目标

针对入门级开发者掌握Git的核心基础知识，分为四个主要部分：基础概念、基础配置、基础命令和实践配置。

- **基础概念**：  
  例如，工作目录（用于临时文件）、暂存区（暂存更改区域）和仓库（版本存储库）。

- **基础配置**：  
  核心设置包括：
  - `.gitignore`文件配置（忽略不必要的文件）
  - username和useremail配置（用于提交信息关联）
  - 查看配置（检查当前设置）

- **基础命令**：  
  常用操作命令如下：
  - `git init` - 初始化新仓库
  - `git clone` - 克隆现有仓库
  - `git add` - 添加文件到暂存区
  - `git commit` - 提交更改
  - `git status` - 查看工作区和暂存区状态
  - `git log` - 查看提交历史
  - `git diff` - 查看更改内容
  - `git pull` - 拉取远程更改
  - `git push` - 推送本地更改

- **实践配置**：  
  进阶配置实践，如配置SSH（安全连接设置）、配置账号密码、配置个人访问令牌（例如GitHub的PAT）。

# 过程

## 基础概念

现代版本控制系统（以Git为代表）基于三个相互关联的核心组件，共同构成完整的版本管理生命周期：**工作目录、暂存区与仓库**。三者形成连续的变更管理流水线，支撑从本地修改到版本固化的全流程。

### 工作目录（Working Directory）

定义：项目文件在本地文件系统的实体存储位置  
核心价值：  
• 承载项目文件的实时工作状态  
• 提供直接操作接口（创建/修改/删除文件）  

运作特征：  
• 文件状态分为未跟踪（untracked）或已修改（modified）  
• 必须显式执行命令才能将变更纳入版本管理  
• 物理层面表现为单文件目录树结构  

### 暂存区（Staging Area）

定义：工作目录与仓库间的逻辑缓冲区  
核心价值：  
• 精准筛选待提交的变更集合  
• 构建具有逻辑一致性的提交单元  

运作特征：  
• 通过git add指令将文件提升至已暂存（staged）状态  
• 支持部分提交与变更预审机制  
• 技术实现依赖于.git/index元数据文件  

### 仓库（Repository）

定义：存储项目全量历史版本及元数据的数据库  
核心价值：  
• 永久保存版本化文件快照  
• 实现历史回溯、分支管理与协作同步  

拓扑架构：  
• 本地仓库：.git目录存储所有版本元数据  
• 远程仓库：网络化共享存储节点（如GitHub）  


## 基础配置

### gitignore配置

gitignore 文件是 Git 版本控制系统的核心配置文件之一，是存储于 Git 仓库根目录的文本文件，它指示 Git 忽略特定文件或目录的跟踪。

#### 语法规则

语法规则是配置 .gitignore 的基础，其核心形式包括：

| 规则类型       | 示例             | 作用描述                     |
|----------------|------------------|------------------------------|
| 直接匹配文件名 | `secret.txt`     | 忽略所有同名文件             |
| 目录忽略       | `logs/`          | 忽略整个目录                 |
| 通配符匹配     | `*.log`          | 忽略所有 `.log` 后缀文件     |
| 递归忽略       | `**/tmp`         | 忽略所有层级的 `tmp` 目录    |
| 取反规则       | `!important.log` | 不忽略指定文件（例外规则）   |
| 注释           | `# 数据库文件`    | 添加说明性注释               |

运行 git status 命令，被忽略的文件将不再显示为 "Untracked"。

#### 特殊使用

##### 取消已commit的文件

git rm --cached config.env  后面跟上需要执行的gitignore配置命令
重新 add commit即可

注意，这样就可以不用删除文件来清楚缓存了，但也要注意，以前的历史记录还是会存在，如果想删除可以问ai，但是不建议。

### git的基础配置

#### 用户身份配置

git config --global user.name  # 查看全局用户名  
git config --global user.email   # 查看全局邮箱

git config --global user.name "Your Name"  
git config --global user.email "your.email@example.com"


git config --list  # 显示所有生效配置

git config --unset user.name          # 删除当前仓库用户名
git config --global --unset user.email # 删除全局邮箱

#### 配置优先级

Git配置分为三级，优先级从高到低：

| **层级**       | **配置文件路径**                     | **生效范围**          | **配置命令**             |
|----------------|------------------------------------|----------------------|-------------------------|
| **本地配置**    | 项目目录下的 `.git/config`         | 仅当前仓库           | `git config --local`    |
| **全局配置**    | 用户主目录的 `~/.gitconfig`        | 当前用户所有仓库      | `git config --global`   |
| **系统配置**    | 系统级的 `/etc/gitconfig`          | 所有用户             | `git config --system`   |

> 📌 **优先级规则**：  
> **本地配置 > 全局配置 > 系统配置**  
> （例如：仓库内设置的邮箱会覆盖全局邮箱）


## 基础语法


#### **一、版本库初始化与克隆**  
1. **`git init`**  
   - **核心功能**：初始化本地Git仓库目录结构  
   - **应用场景**：新建项目版本控制初始配置  
   - **操作示例**：  
     ```bash
     cd project_directory/  # 进入目标目录
     git init               # 生成.git元数据存储目录
     ```

2. **`git clone`**  
   - **核心功能**：完整复制远程版本库至本地（含历史记录）  
   - **应用场景**：协作场景下获取现有代码库  
   - **操作示例**：  
     ```bash
     git clone https://github.com/user/repository.git
     ```

#### **二、本地变更管理**  
3. **`git add`**  
   - **核心功能**：遴选工作区变更至暂存区（Staging Area）  
   - **应用场景**：提交前筛选版本更新内容  
   - **操作示例**：  
     ```bash
     git add specific_file.ext  # 添加指定文件
     git add .                  # 添加当前目录全量变更
     ```

4. **`git commit`**  
   - **核心功能**：持久化暂存区变更至本地版本库  
   - **关键参数**：`-m` 直连提交描述信息  
   - **操作示例**：  
     ```bash
     git commit -m "修复用户认证逻辑缺陷"
     ```

---

#### **三、版本状态检视**  
5. **`git status`**  
   - **核心功能**：实时反馈工作区/暂存区文件状态  
   - **状态标识**：  
     - **红色文本**：未跟踪或未暂存变更  
     - **绿色文本**：已暂存待提交变更  
   ```bash
   git status  # 建议关键操作前执行
   ```

6. **`git log`**  
   - **核心功能**：检索当前分支版本演进历史  
   - **高效参数**：  
     ```bash
     git log --oneline  # 精简输出模式（提交哈希+描述）
     ```

7. **`git diff`**  
   - **核心功能**：对比工作区与暂存区内容差异  
   - **扩展对比**：  
     ```bash
     git diff HEAD~1  # 比较当前提交与前次版本
     ```

---

#### **四、跨域协作指令**  
8. **`git pull`**  
   - **技术本质**：`git fetch`（远程变更获取） + `git merge`（分支合并）  
   - **应用场景**：同步远程仓库最新演进  
   - **操作示例**：  
     ```bash
     git pull origin main  # 拉取origin远端main分支
     ```

9. **`git push`**  
   - **核心功能**：向远程仓库推送本地演进历史  
   - **首次关联**：  
     ```bash
     git push -u origin main  # -u建立本地与远程分支追踪
     ```

---

### **标准化工作流速查**  
| 操作阶段       | 规范流程                     |  
|----------------|------------------------------|  
| 日常开发       | `status` → `add` → `commit` → `push` |  
| 协作集成       | `pull` → 冲突处理 → `add` → `commit` → `push` |  
| 审查准备       | `diff` → `log` → `status`     |
