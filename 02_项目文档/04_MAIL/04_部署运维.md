# s-pay-mall - 部署运维

---

## 1. 编译打包

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 1.8 |
| Maven | 3.11.0+ |
| Docker | 20.10+ |
| Docker Compose | 2.0+ |

### Maven 构建

```bash
# 开发环境（默认）
mvn clean package -DskipTests

# 指定环境
mvn clean package -P dev -DskipTests    # 开发
mvn clean package -P test -DskipTests   # 测试
mvn clean package -P prod -DskipTests   # 生产
```

### Maven Profiles

| Profile | JVM 参数 | 用途 |
|---------|----------|------|
| dev | -Xms1G -Xmx1G -XX:+UseG1GC | 开发环境（默认激活） |
| test | -Xms1G -Xmx1G -XX:+UseG1GC | 测试环境 |
| prod | -Xms6G -Xmx6G -XX:+UseG1GC | 生产环境 |

## 2. 部署方式

### 2.1 基础环境部署（MySQL + Redis）

```bash
cd docs/dev-ops/
docker-compose -f docker-compose-environment.yml up -d
```

启动的服务：

| 服务 | 容器名 | 端口映射 | 说明 |
|------|--------|----------|------|
| MySQL 8.0.32 | mysql | 13306 → 3306 | 密码: 123456 |
| phpMyAdmin | phpmyadmin | 8899 → 80 | 数据库管理 |
| Redis 6.2 | redis | 16379 → 6379 | 缓存服务 |
| Redis Commander | redis-admin | 8081 → 8081 | Redis 管理界面（admin/admin） |

数据库会自动执行 `docs/dev-ops/mysql/sql/s-pay-mall.sql` 初始化脚本。

### 2.2 应用服务部署

```bash
cd docs/dev-ops/
docker-compose -f docker-compose-app.yml up -d
```

| 配置项 | 值 |
|--------|-----|
| 镜像 | system/s-pay-mall:1.0 |
| 容器名 | s-pay-mall |
| 端口 | 8091 → 8091 |
| 时区 | PRC |
| 日志 | JSON文件，max 10m × 3 |

### 2.3 本地开发运行

直接运行 `Application.java` 主类，默认激活 `dev` profile，服务端口 **8092**。

## 3. 配置项

| tomcat.threads.min-spare | 10 | 最小空闲线程 |

## 5. 内网穿透

支付宝回调需要公网可达的 URL，开发环境需配置内网穿透：

| 工具 | 说明 |
|------|------|
| natapp | 配置参考回调地址：`http://xfg-studio.natapp1.cc` |

回调地址映射到本地服务的 `/api/v1/alipay/pay_notify`。
