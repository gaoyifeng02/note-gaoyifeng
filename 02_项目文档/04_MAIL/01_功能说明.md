# s-pay-mall - 支付商城系统

> 基于 DDD（领域驱动设计）架构的支付商城系统，集成支付宝沙箱支付与微信公众号登录。
> 源码仓库：`D:\workspace\xfg\s-pay-mall-ddd-qixi_day\s-pay-mall-ddd-qixi_day`

---

## 1. 服务定位与职责

s-pay-mall 是一个**教学级 DDD 支付商城系统**，核心职责：

- **商品下单与支付**：用户选择商品后创建订单，通过支付宝沙箱完成支付
- **微信公众号登录**：支持微信扫码二维码登录
- **订单生命周期管理**：覆盖创建、待支付、支付成功、交易完成、超时关单全流程
- **支付回调处理**：接收支付宝异步回调，验证签名并更新订单状态
- **定时任务补偿**：主动查询未回调订单、超时未支付订单自动关单

## 3. 核心功能列表

### 3.1 订单子系统（order 子域）

| 功能 | 说明 |
|------|------|
| 创建订单 | 用户通过购物车选择商品，系统生成16位随机订单号 |
| 防重复下单 | 查询同一用户+商品的未支付订单，避免重复创建 |
| 掉单恢复 | 对 CREATE 状态的订单重新生成支付单，不创建新订单 |
| 支付单生成 | 调用支付宝 SDK 生成 PC 网页支付表单 |
| 支付回调 | 接收支付宝异步通知，RSA2 验签后更新订单状态 |
| 支付成功事件 | 订单支付成功后通过 EventBus 发布事件 |
| 超时关单 | 定时扫描超时30分钟未支付订单，自动关闭 |
| 回调补偿 | 定时扫描已支付但未收到回调的订单，主动查询支付宝 |

### 3.2 认证子系统（auth 子域）

| 功能 | 说明 |
|------|------|
| 生成登录二维码 | 调用微信公众号 API 创建带参数的临时二维码 |
| 扫码登录回调 | 接收微信 SCAN 事件，保存登录状态 |
| 登录状态轮询 | 前端通过 ticket 轮询检查登录状态 |
| 登录成功通知 | 扫码成功后发送微信公众号模板消息 |

## 4. 支持的业务场景

```
场景1：商品下单支付
  用户选择商品 → 创建订单 → 生成支付宝支付表单 → 用户完成支付 → 回调更新订单 → 触发后续业务（发货/充值）

场景2：掉单/未支付订单恢复
  用户已有未支付订单 → 返回原支付链接 / 重新生成支付单

场景3：微信扫码登录
  前端请求二维码 → 用户扫码 → 微信回调保存状态 → 前端轮询获取 token

场景4：超时订单处理
  定时任务扫描 → 关闭超时未支付订单

场景5：支付回调丢失补偿
  定时任务扫描 → 主动查询支付宝交易状态 → 补偿更新
```
