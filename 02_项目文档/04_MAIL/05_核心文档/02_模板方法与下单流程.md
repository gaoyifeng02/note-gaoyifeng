## 1. 模板方法模式设计

`AbstractOrderService` 定义了下单的标准流程骨架，将可变步骤延迟到子类实现：

```java
public abstract class AbstractOrderService implements IOrderService {

    protected final IOrderRepository repository;
    protected final IProductPort productPort;

    // 模板方法：定义标准下单流程
    @Override
    public PayOrderEntity createOrder(ShopCartEntity shopCartEntity) throws Exception {
        // Step 1: 查询未支付订单（防重复 + 掉单恢复）
        // ... 固定逻辑

        // Step 2: 查询商品信息
        // ... 固定逻辑

        // Step 3: 构建聚合根
        // ... 固定逻辑

        // Step 4: 保存订单（抽象方法 - 子类实现）
        this.doSaveOrder(orderAggregate);

        // Step 5: 创建支付单（抽象方法 - 子类实现）
        PayOrderEntity payOrderEntity = this.doPrepayOrder(...);

        return payOrderEntity;
    }

    // 抽象方法：保存订单
    protected abstract void doSaveOrder(CreateOrderAggregate orderAggregate);

    // 抽象方法：预支付订单
    protected abstract PayOrderEntity doPrepayOrder(String userId, String productId,
        String productName, String orderId, BigDecimal totalAmount) throws AlipayApiException;
}
```

## 2. 子类实现：支付宝支付

```java
@Service
public class OrderService extends AbstractOrderService {

    @Resource
    private AlipayClient alipayClient;

    @Override
    protected void doSaveOrder(CreateOrderAggregate orderAggregate) {
        repository.doSaveOrder(orderAggregate);
    }

    @Override
    protected PayOrderEntity doPrepayOrder(...) throws AlipayApiException {
        // 1. 构建支付宝请求
        AlipayTradePagePayRequest request = new AlipayTradePagePayRequest();
        request.setNotifyUrl(notifyUrl);
        request.setReturnUrl(returnUrl);

        JSONObject bizContent = new JSONObject();
        bizContent.put("out_trade_no", orderId);
        bizContent.put("total_amount", totalAmount.toString());
        bizContent.put("subject", productName);
        bizContent.put("product_code", "FAST_INSTANT_TRADE_PAY");
        request.setBizContent(bizContent.toString());

        // 2. 调用支付宝 SDK 生成支付表单
        String form = alipayClient.pageExecute(request).getBody();

        // 3. 更新订单状态为 PAY_WAIT
        PayOrderEntity payOrderEntity = new PayOrderEntity();
        payOrderEntity.setOrderId(orderId);
        payOrderEntity.setPayUrl(form);
        payOrderEntity.setOrderStatus(OrderStatusVO.PAY_WAIT);
        repository.updateOrderPayInfo(payOrderEntity);

        return payOrderEntity;
    }
}
```

## 3. 防重复下单与掉单恢复

`createOrder()` 方法的 Step 1 实现了两种恢复策略：

```
createOrder(ShopCartEntity)
    │
    ├─ 查询 queryUnPayOrder(userId, productId)
    │
    ├─ 情况A: 存在 PAY_WAIT 状态订单
    │   └─ 直接返回原支付链接（用户可继续支付）
    │
    ├─ 情况B: 存在 CREATE 状态订单
    │   └─ 重新生成支付单（不创建新订单，避免重复）
    │
    └─ 情况C: 无未支付订单
        └─ 正常创建新订单 + 支付单
```

### 为什么需要这个机制？

- **PAY_WAIT 恢复**：用户已生成支付单但未完成支付，返回原链接避免重复下单
- **CREATE 恢复**：订单已创建但支付单生成失败（如支付宝接口超时），从创建状态重新生成支付单
- **索引保障**：`idx_user_id_product_id` 联合索引确保查询效率

## 4. 聚合根设计

```java
@Data
@Builder
public class CreateOrderAggregate {
    private String userId;
    private ProductEntity productEntity;
    private OrderEntity orderEntity;

    // 静态工厂方法：生成订单实体
    public static OrderEntity buildOrderEntity(String productId, String productName) {
        return OrderEntity.builder()
                .productId(productId)
                .productName(productName)
                .orderId(RandomStringUtils.randomNumeric(16))  // 16位随机数字
                .orderTime(new Date())
                .orderStatus(OrderStatusVO.CREATE)
                .build();
    }
}
```

聚合根将一次下单操作需要的所有数据（用户、商品、订单）封装在一起，确保数据一致性。

## 5. DB + Redis 双写

`OrderRepository.doSaveOrder()` 实现了双写：

```java
@Override
public void doSaveOrder(CreateOrderAggregate orderAggregate) {
    // 1. 写 MySQL
    PayOrder order = new PayOrder();
    // ... 属性映射
    orderDao.insert(order);

    // 2. 写 Redis
    redisService.setValue(PayOrder.cacheKey(userId, orderId), order);
}
```

缓存 Key 由 PO 对象提供静态方法生成，确保 Key 格式统一。
