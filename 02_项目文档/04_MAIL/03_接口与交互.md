# s-pay-mall - 接口与交互

---

## 1. 对外 API 接口

### 1.1 支付接口（AliPayController）

**基础路径**：`/api/v1/alipay/`

#### 创建支付订单

```
POST /api/v1/alipay/create_pay_order
Content-Type: application/json

请求体：
{
    "userId": "1001",
    "productId": "100001"
}

成功响应：
{
    "code": "0000",
    "info": "成功",
    "data": "<form>...</form>"    // 支付宝支付表单 HTML
}

失败响应：
{
    "code": "E0001",
    "info": "系统异常"
}
```

#### 支付回调通知

```
POST /api/v1/alipay/pay_notify
Content-Type: application/x-www-form-urlencoded

参数（支付宝异步通知标准参数）：
  trade_status: TRADE_SUCCESS
  out_trade_no: 订单号
  trade_no: 支付宝交易凭证号
  total_amount: 交易金额
  gmt_payment: 付款时间
  sign: RSA2 签名
  ...其他支付宝参数

返回值：
  "success"  // 处理成功
  "false"    // 处理失败
```

### 1.2 登录接口（LoginController）

**基础路径**：`/api/v1/login/`

#### 获取微信二维码 Ticket

```
GET /api/v1/login/weixin_qrcode_ticket

成功响应：
{
    "code": "0000",
    "info": "成功",
    "data": "gQH47joAA..."    // 微信二维码 ticket
}
```

#### 轮询检查登录状态

```
GET /api/v1/login/check_login?ticket=xxx

已登录响应：
{
    "code": "0000",
    "info": "成功",
    "data": "openid_xxx"      // 用户 openid
}

未登录响应：
{
    "code": "E0002",
    "info": "未登录"
}
```

### 1.3 微信公众号门户（WeixinPortalController）

**基础路径**：`/api/v1/weixin/portal/`

#### 验签（GET）

```
GET /api/v1/weixin/portal/receive?signature=xxx&timestamp=xxx&nonce=xxx&echostr=xxx

返回值：echostr（验签通过时）
```

#### 消息回调（POST）

```
POST /api/v1/weixin/portal/receive
Content-Type: application/xml

请求体：微信 XML 消息
<xml>
  <MsgType>event</MsgType>
  <Event>SCAN</Event>
  <Ticket>xxx</Ticket>
  <FromUserName>openid</FromUserName>
</xml>

返回值：XML 格式回复消息
```

## 2. 统一响应结构

```java
Response<T> {
    String code;    // 响应码
    String info;    // 响应信息
    T data;         // 业务数据
}
```

### 响应码枚举（ResponseCode）

| code | 含义 |
|------|------|
| 0000 | 成功 |
| E0001 | 系统异常 |
| E0002 | 未登录 |

### Constants 常量

| 常量 | 值 | 说明 |
|------|-----|------|
| ResponseCode.SUCCESS | "0000" | 成功 |
| ResponseCode.UN_ERROR | "E0001" | 系统异常 |
| ResponseCode.NO_LOGIN | "E0002" | 未登录 |

## 3. 数据库交互

### 3.1 数据库信息

| 项目 | 值 |
|------|-----|
| 数据库名 | s-pay-mall |
| 字符集 | utf8mb4 |
| 引擎 | InnoDB |

### 3.2 pay_order 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT UNSIGNED AUTO_INCREMENT | 自增主键 |
| user_id | VARCHAR(32) | 用户ID |
| product_id | VARCHAR(16) | 商品ID |
| product_name | VARCHAR(64) | 商品名称 |
| order_id | VARCHAR(16) | 订单ID（唯一索引） |
| order_time | DATETIME | 下单时间 |
| total_amount | DECIMAL(8,2) | 订单金额 |
| status | VARCHAR(32) | 订单状态 |
| pay_url | VARCHAR(2014) | 支付信息（HTML表单） |
| pay_time | DATETIME | 支付时间 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间（自动更新） |

**索引**：
- PRIMARY KEY: `id`
- UNIQUE KEY: `uq_order_id` (`order_id`)
- KEY: `idx_user_id_product_id` (`user_id`, `product_id`)

### 3.3 订单状态流转

```
CREATE → PAY_WAIT → PAY_SUCCESS → DEAL_DONE
   │                                     ↑
   └─────────── CLOSE（超时关单）──────────┘
```

| 状态 | 说明 |
|------|------|
| CREATE | 创建完成（调单后从此状态重新发起支付） |
| PAY_WAIT | 等待支付（已生成支付单） |
| PAY_SUCCESS | 支付成功（收到回调或补偿查询确认） |
| DEAL_DONE | 交易完成（商品发货完成） |
| CLOSE | 超时关单（超时未支付） |

## 5. 外部服务依赖

### 5.1 支付宝沙箱

| 配置项 | 说明 |
|--------|------|
| gatewayUrl | `https://openapi-sandbox.dl.alipaydev.com/gateway.do` |
| notify_url | 支付回调地址（需内网穿透） |
| return_url | 支付完成跳转地址 |
| 签名方式 | RSA2 |

交互方式：通过 AlipayClient SDK 同步调用

### 5.2 微信公众号

| 配置项 | 说明 |
|--------|------|
| originalid | 公众号原始ID |
| app-id | 应用ID |
| app-secret | 应用密钥 |
| template_id | 模板消息ID |

交互方式：通过 Retrofit2 HTTP 客户端调用微信 API

### 5.3 商品服务（ProductRPC）

当前为 Mock 实现，硬编码返回商品信息。未来可替换为真实 RPC 调用。

## 6. 内部事件

| 事件 | Topic | 生产者 | 消费者 | 数据 |
|------|-------|--------|--------|------|
| PaySuccessMessageEvent | pay_success | OrderRepository | OrderPaySuccessListener | `{tradeNo, userId}` |

实现方式：Guava EventBus（进程内），通过 `@Subscribe` 注解监听。
