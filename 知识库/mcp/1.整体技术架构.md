AI MCP Gateway 是一个基于 Model Context Protocol (MCP) 的智能网关服务系统，专为 AI 智能体应用提供高性能、可扩展的 MCP 实时通信基础设施。解决 AI 应用与各种数据源、工具和服务之间的连接复杂性，提供统一的接口和协议标准。

整个实现过程是以 MCP 协议 JSON-RPC2 通信结构，设计实现流式传输 Streamable HTTP 框架，完成 MCP 网关接口协议解析和消息处理。

在消息处理过程中，则对注册进来的 http/rpc/... 接口进行调用。这部分会包括接口的描述性信息获取以及会话执行调用。

- 网关服务注册阶段
	- 用户向触发器（网关入口）发起网关服务注册请求。
	- 触发器将请求透传至编排层，完成网关配置数据的录入。
	- 编排层调用领域层录入具体的 http/rpc 协议数据。
	- 领域层在“通信核心子域”内构建服务映射，并在基础设施层分别完成 http mapper 和 rpc mapper 的持久化。
	- 注册完成后，逐级返回成功信号至用户。
- 建立 MCP 连接（SSE）阶段
	- 用户发起获取 SSE 连接的请求（get {gatewayId}/mcp/sse）。
	- 编排层向领域层发出建立 MCP 连接的建议。
	- 领域层通过基础设施层校验 sessionId 以及网关映射数据的合法性。
	- 校验通过后，领域层创建并保存 Session 会话 ID。
	- 领域层反馈消息接收地址（endpoint）给编排层。
	- 触发器最终向用户返回包含 endpoint 信息的 SSE 事件，建立长连接通信。
- 响应 MCP 消息与业务处理阶段
	- 用户通过 POST 请求发送 MCP 指令（如 initialize、tools/list、tools/call）。
	- 编排层首先调用领域层进行网关服务和 sessionId 的有效性校验。
	- 编排层内部对请求进行 MCPJsonRpc2 协议解析，识别具体的业务指令。
	- 根据解析出的不同指令，领域层执行相应逻辑：
	- 若为 initialize：从基础设施层获取并返回接口描述。
	- 若为 tools/list：从基础设施层获取接口细节（含入参描述）并构建工具列表。
	- 若为 tools/call：通过基础设施层发起真正的 http/rpc 远程服务调用。
	- 业务处理结果由编排层封装后，经触发器返回给用户，完成一次完整的 MCP 指令交互。
