## 一、Result 响应封装体系

### 架构设计

```
ResultConstant (接口)
    │ getCode() / getInfo()
    │
    ├── ResponseCode (枚举) ← 标准响应码，所有项目共用
    │     SUCCESS / FAIL / UNAUTHORIZED / ...
    │
    └── 业务自定义枚举 ← 各项目按需扩展
          OrderResponseCode / PayResponseCode / ...

Result<T> ← 统一响应包装
    ├── code: String    (来自 ResultConstant)
    ├── info: String    (来自 ResultConstant)
    └── data: T         (业务数据)

PageResult<T> ← 分页数据封装
    ├── total: Long
    ├── pageNo: Integer
    ├── pageSize: Integer
    └── result: Collection<T>
```

### 调用链路

```
Controller → Result.success(data)
           → Result.fail(ResponseCode.PARAM_ERROR)
           → Result.fail(自定义枚举, "覆盖信息")

HTTP 响应示例：
{
    "code": "0000",
    "info": "成功",
    "data": { ... }
}
```

## 二、策略模式（tree 包）

### 类图关系

```
StrategyHandler<T,D,R>          ← 策略执行接口
    │ apply(request, context)
    │ DEFAULT (空实现)
    │
    └── 具体策略实现
          AlipayHandler / WechatHandler / ...

StrategyMapper<T,D,R>           ← 策略选择接口
    │ get(request, context)
    │
    └── AbstractStrategyRouter   ← 路由器（同时实现 Handler + Mapper）
          │ router(request, context)  ← 入口方法
          │
          └── AbstractMultiThreadStrategyRouter  ← 多线程版本
                │ multiThread()  ← 异步加载
                │ doApply()      ← 业务处理
                │ applyBefore / applyAfter / applyAfterException
```

### 执行流程

```
router(request, context)
    │
    ├── get(request, context)  →  获取策略处理器
    │     │
    │     ├── 返回具体 Handler  →  执行 handler.apply()
    │     └── 返回 null         →  执行 defaultStrategyHandler
    │
    └── 返回 Result
```

## 三、责任链模式（link 包）

### 类图关系

```
ILogicHandler<T,D,R>            ← 处理器接口
    │ apply()            ← 主逻辑（必须实现）
    │ applyBefore()      ← 前置处理
    │ applyAfter()       ← 后置处理
    │ applyAfterException() ← 异常处理
    │
    ├── next()    → 继续
    ├── stop()    → 中断
    └── jump()    → 跳过

BusinessLinkedList<T,D,R>       ← 链路容器（继承 LinkedList）
    │ apply(request, context)   ← 驱动所有处理器依次执行
    │
    └── LinkArmory<T,D,R>      ← 链路组装工厂
          new LinkArmory("链名", handler1, handler2, handler3)

DynamicContext                  ← 上下文
    ├── proceed: boolean        ← 是否继续
    ├── jump: boolean           ← 是否跳过
    └── dataObjects: Map        ← 共享数据
```

### 执行流程

```
BusinessLinkedList.apply(request, context)
    │
    for (handler in handlers):
    │
    ├── handler.applyBefore(request, context)
    │     └── if !proceed → return（链路中断）
    │
    ├── if jump → continue（跳过当前 handler）
    │
    ├── handler.apply(request, context)
    │     └── if !proceed → return（链路中断）
    │
    ├── handler.applyAfter(request, context, result)
    │
    └── catch → handler.applyAfterException(request, context, e)
```

## 五、全局异常处理体系

### 架构设计

```
各层抛出异常
    │
    ├── domain: throw new BaseException(ResponseCode.XXX)
    ├── case: 捕获领域异常，转换为业务语义异常
    ├── infrastructure: catch 技术异常 → 包装为 BaseException
    │
    └── → 最终全部到达 trigger 层
              │
              GlobalExceptionHandler (@RestControllerAdvice)
              │
              ├── BaseException        → warn 日志 + 200 + 业务错误码
              ├── IllegalArgumentException → warn 日志 + 400
              ├── Exception (兜底)     → error 日志 + 500
              │
              └── 统一返回 Result<Void>
```

### 异常分层职责

| 层 | 职责 | 示例 |
|---|---|---|
| types | 定义 BaseException、ResponseCode | 框架基础 |
| domain | 抛出领域语义异常 | `throw new BaseException(NOT_FOUND)` |
| infrastructure | 技术异常转领域异常 | `catch SQLException → BaseException(INTERNAL_ERROR)` |
| case | 可选的异常语义转换 | 多领域组合时的统一翻译 |
| trigger | HTTP 异常 → JSON 响应 | GlobalExceptionHandler |

### 使用方式

```java
// 直接使用
throw new BaseException(ResponseCode.NOT_FOUND);
throw new BaseException(ResponseCode.PARAM_ERROR, "用户名不能为空");

// 业务项目可继承扩展
public class BizException extends BaseException {
    public BizException(ResponseCode code) { super(code); }
}
```

## 六、自动装配机制

### 设计原理

```
META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
    │ 注册
    └── JavaBaseAutoConfiguration (@AutoConfiguration)
            │ @ConditionalOnWebApplication  ← 仅 Web 环境生效
            │
            └── @Bean GlobalExceptionHandler  ← 自动注册到 Spring 容器
```

### 引入方式

业务项目只需一行依赖：

```xml
<dependency>
    <groupId>com.gaoyifeng</groupId>
    <artifactId>java-base-starter</artifactId>
    <version>1.0.0-SNAPSHOT</version>
</dependency>
```

starter 传递依赖链：
```
java-base-starter
    ├── java-base-trigger     → GlobalExceptionHandler + 自动装配
    │       ├── java-base-case → java-base-domain → java-base-api → java-base-types
    │       └── java-base-types
    └── java-base-infrastructure → MySQL, Redis, MyBatis
            └── java-base-domain
```

无需 @ComponentScan、无需手动配置，引入即生效。
