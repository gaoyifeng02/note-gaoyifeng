## 1. 事件驱动架构

### 1.1 事件总线选型

当前使用 **Guava EventBus**（进程内事件总线），优势：

- 零外部依赖，适合教学和小型项目
- 同步调用，调试方便
- 未来可无缝切换到 RocketMQ/Kafka

### 1.2 事件定义

```java
// 事件基类
public abstract class BaseEvent<T> {
    public abstract EventMessage<T> buildEventMessage(T data);
    public abstract String topic();
}

// 支付成功事件
@Component
public class PaySuccessMessageEvent extends BaseEvent<PaySuccessMessageEvent.PaySuccessMessage> {

    @Override
    public String topic() {
        return "pay_success";
    }

    @Data
    @Builder
    public static class PaySuccessMessage {
        private String userId;
        private String tradeNo;
    }
}
```

### 1.3 事件发布

```java
// OrderRepository.changeOrderPaySuccess()
@Override
public void changeOrderPaySuccess(String orderId) {
    // 1. 更新 DB
    PayOrder order = new PayOrder();
    order.setOrderId(orderId);
    order.setStatus(OrderStatusVO.PAY_SUCCESS.getCode());
    orderDao.changeOrderPaySuccess(order);

    // 2. 发布事件
    BaseEvent.EventMessage<PaySuccessMessage> eventMessage =
        paySuccessMessageEvent.buildEventMessage(
            PaySuccessMessage.builder().tradeNo(orderId).build()
        );
    eventBus.post(JSON.toJSONString(eventMessage.getData()));
}
```

### 1.4 事件消费

```java
@Slf4j
@Component
public class OrderPaySuccessListener {

    @Subscribe
    public void handleEvent(String paySuccessMessage) {
        log.info("收到支付成功消息，可以做接下来的事情了【发货、充值、开会员】{}", paySuccessMessage);
    }
}
```

### 1.5 EventBus 注册

```java
// GuavaConfig.java
@Configuration
public class GuavaConfig {

    @Bean
    public EventBus eventBus(Listener[] listeners) {
        EventBus eventBus = new EventBus();
        for (Listener listener : listeners) {
            eventBus.register(listener);
        }
        return eventBus;
    }
}
```

> 所有带 `@Subscribe` 注解的 Listener 都会被自动注册到 EventBus。

## 2. 支付补偿机制

### 2.1 问题：回调丢失

支付完成后，支付宝通过异步回调通知商户。但以下场景可能导致回调丢失：

- 网络超时
- 回调地址不可达
- 回调处理异常

### 2.2 解决方案：主动查询补偿

```
NoPayNotifyOrderJob（每 3 秒执行）
    │
    ├─ 1. 查询未收到回调的订单（DB 查询 PAY_WAIT 状态订单）
    │
    └─ 2. 遍历每个订单，主动查询支付宝交易状态
        │
        ├─ AlipayTradeQueryRequest → alipayClient.execute()
        │
        └─ 判断返回码：
            ├─ "10000" → 交易成功 → orderService.changeOrderPaySuccess()
            └─ 其他 → 等待下次轮询
```

```java
@Scheduled(cron = "0/3 * * * * ?")
public void exec() {
    List<String> orderIds = orderService.queryNoPayNotifyOrder();
    if (null == orderIds || orderIds.isEmpty()) return;

    for (String orderId : orderIds) {
        AlipayTradeQueryRequest request = new AlipayTradeQueryRequest();
        AlipayTradeQueryModel bizModel = new AlipayTradeQueryModel();
        bizModel.setOutTradeNo(orderId);
        request.setBizModel(bizModel);

        AlipayTradeQueryResponse response = alipayClient.execute(request);
        if ("10000".equals(response.getCode())) {
            orderService.changeOrderPaySuccess(orderId);
        }
    }
}
```

### 2.3 超时关单

```
TimeoutCloseOrderJob（每 10 分钟执行）
    │
    ├─ 查询超时30分钟未支付订单（PAY_WAIT 且 超过30分钟）
    │
    └─ 逐个关闭：orderService.changeOrderClose(orderId)
        └─ 更新状态为 CLOSE
```

```java
@Scheduled(cron = "0 0/10 * * * ?")
public void exec() {
    List<String> orderIds = orderService.queryTimeoutCloseOrderList();
    if (null == orderIds || orderIds.isEmpty()) return;

    for (String orderId : orderIds) {
        boolean status = orderService.changeOrderClose(orderId);
        log.info("定时任务，超时30分钟订单关闭 orderId:{} status:{}", orderId, status);
    }
}
```

## 3. 完整订单状态机

```
                    创建订单
                      │
                      ▼
                 ┌─────────┐
                 │ CREATE  │
                 └────┬────┘
                      │ 生成支付单
                      ▼
                ┌───────────┐     支付回调 / 补偿查询
                │ PAY_WAIT  │──────────────────────────┐
                └─────┬─────┘                          │
                      │                                │
            超时30分钟未支付                              │
                      │                                │
                      ▼                                ▼
               ┌───────────┐                    ┌──────────────┐
               │   CLOSE   │                    │ PAY_SUCCESS  │
               └───────────┘                    └──────┬───────┘
                                                     │
                                                     │ 后续业务处理
                                                     ▼
                                              ┌───────────┐
                                              │ DEAL_DONE │
                                              └───────────┘
```

| 状态转换 | 触发者 | 说明 |
|----------|--------|------|
| → CREATE | createOrder() | 下单成功，生成16位订单号 |
| CREATE → PAY_WAIT | doPrepayOrder() | 支付宝支付表单生成成功 |
| PAY_WAIT → PAY_SUCCESS | payNotify() / NoPayNotifyOrderJob | 收到回调或补偿查询确认 |
| PAY_WAIT → CLOSE | TimeoutCloseOrderJob | 超时30分钟自动关单 |
| PAY_SUCCESS → DEAL_DONE | 后续业务 | 发货/充值完成后（预留） |
