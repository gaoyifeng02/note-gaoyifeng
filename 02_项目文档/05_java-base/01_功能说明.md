## 一、服务定位与职责

java-base 是所有 Java 项目的**基础模板项目**，提供统一的基础组件和设计模式，所有新建 Java 项目必须基于此项目。

核心职责：
- 定义统一的响应封装体系（Result / ResponseCode / PageResult）
- 提供可复用的设计模式骨架（策略模式、责任链模式）
- 建立 DDD 分层架构的标准模块结构

---

## 二、核心功能列表

| 功能 | 包路径 | 说明 |
|------|--------|------|
| 统一响应封装 | `types.common.Result` | 泛型响应包装类，支持成功/失败/自定义响应码 |
| 标准响应码 | `types.common.ResponseCode` | 通用响应码枚举（成功、失败、未授权等） |
| 响应码契约 | `types.common.ResultConstant` | 响应码接口，业务可扩展自定义响应码 |
| 分页封装 | `types.common.PageResult` | 分页数据封装，作为 Result.data 使用 |
| 策略模式 | `types.design.tree` | AbstractStrategyRouter 策略路由框架 |
| 责任链模式 | `types.design.link` | BusinessLinkedList 责任链框架 |

## 四、模块结构

```
java-base/
├── java-base-api/              # API 接口层
├── java-base-app/              # 应用启动层
├── java-base-case/             # 用例编排层
├── java-base-domain/           # 领域层
├── java-base-infrastructure/   # 基础设施层
├── java-base-trigger/          # 触发器层
└── java-base-types/            # 通用类型层（核心：Result、设计模式）
```

其中 `java-base-types` 是最核心的模块，包含所有基础组件和设计模式。
