> Obsidian Vault | 个人笔记 + 总项目文档 + AI 知识库

## 项目定位

本项目承担两个核心职责：

1. **个人笔记项目** — 日记、感悟、人生规划
2. **总项目文档中心** — 作为所有项目的知识中枢，其他项目的 `CLAUDE.md` 会指引 AI 来此获取：开发规范、踩坑指南、技术方案、架构设计等

## 关键文档摘要

### AIOServer 策略管控系统（`知识库/工作/策略管理归档.md`）
Go 语言实现的终端策略分发系统。三级缓存（BigCache→Redis→gRPC 回源），位图匹配算法，CAS 乐观锁，灰度发布，推送任务调度。

### MCP 网关项目（`知识库/mcp/`）
Java DDD 架构实现的 MCP 协议网关。将企业内部 HTTP/RPC 接口动态转换为 MCP tools，支持 SSE 长连接，供 Dify/Claude Code 等 AI Agent 调用。

### AI 开发工作流（`知识库/ai-native/方法论/`）
推荐 BDD（行为驱动开发）工作流：feature 确定 → 技术设计 → AI 生成 → BDD 验证。目标：AI 在开发环境完美闭环（Syncthing 同步远程代码→本地编译→测试→部署→查日志→修复循环）。

### 设计模式模板（`知识库/设计模式/`）
- **规则树模式** — `AbstractStrategyRouter` + `StrategyMapper` + `StrategyHandler`，支持多线程异步加载
- **责任链模式** — `ILogicHandler` + `BusinessLinkedList` + `LinkArmory`，支持 next/stop/jump 控制

## AI 使用指南

如果你是 AI 助手，正在阅读此文件，请遵循以下指引：

1. **先看本 README** — 快速了解项目结构和文档位置
2. **按需读取文档** — 不要一次性读取所有文件，根据任务定位到具体路径
3. **知识库 vs 项目文档** — 通用技术问题查 `03_知识库/`，具体项目问题查 `04_项目文档/`
4. **踩坑记录优先** — 遇到技术问题时先查 `03_知识库/09_踩坑指南/`
5. **不要修改隐私目录** — `05_隐私/` 下的内容仅供查阅，不要主动修改或暴露
