## 一、服务定位与职责

java-base 是所有 Java 项目的**基础模板项目**，提供统一的基础组件和设计模式，所有新建 Java 项目必须基于此项目。

核心职责：
- 定义统一的响应封装体系（Result / ResponseCode / PageResult）
- 提供全局异常处理机制（BaseException / GlobalExceptionHandler）
- 提供可复用的设计模式骨架（策略模式、责任链模式）
- 建立 DDD 分层架构的标准模块结构
- 通过 java-base-starter 实现一站式引入，业务项目零配置即可使用所有基础能力

## 二、核心功能列表

| 功能 | 包路径 | 说明 |
|------|--------|------|
| 统一响应封装 | `types.common.Result` | 泛型响应包装类，支持成功/失败/自定义响应码 |
| 标准响应码 | `types.common.ResponseCode` | 通用响应码枚举（成功、失败、未授权等） |
| 响应码契约 | `types.common.ResultConstant` | 响应码接口，业务可扩展自定义响应码 |
| 分页封装 | `types.common.PageResult` | 分页数据封装，作为 Result.data 使用 |
| 异常基类 | `types.exception.BaseException` | 框架级异常基类，支持 ResponseCode 和自定义 ResultConstant |
| 全局异常处理 | `trigger.http.GlobalExceptionHandler` | @RestControllerAdvice 自动捕获异常并转为统一 Result |
| 自动装配 | `trigger.config.JavaBaseAutoConfiguration` | Spring Boot 3.x 自动装配，引入依赖即生效 |
| 策略模式 | `types.design.tree` | AbstractStrategyRouter 策略路由框架 |
| 责任链模式 | `types.design.link` | BusinessLinkedList 责任链框架 |

## 四、模块结构

```
java-base/
├── java-base-api/              # API 接口层
├── java-base-app/              # 应用启动层（演示用，业务项目不需要）
├── java-base-case/             # 用例编排层
├── java-base-domain/           # 领域层
├── java-base-infrastructure/   # 基础设施层（MySQL、Redis、MyBatis）
├── java-base-trigger/          # 触发器层（HTTP 控制器、全局异常处理、自动装配）
├── java-base-types/            # 通用类型层（核心：Result、BaseException、设计模式、工具库）
└── java-base-starter/          # 一站式引入模块（业务项目只需引此依赖）
```

其中 `java-base-types` 是最核心的模块，包含所有基础组件和设计模式。
业务项目只需引入 `java-base-starter` 即可使用全部能力，无需额外配置。
